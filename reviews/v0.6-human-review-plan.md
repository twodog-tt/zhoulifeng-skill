# v0.6 Human Review Plan

## Status

Prepared, not completed.

## Objective

Review public readiness after v0.5.0. The review should verify that the Skill is clear to install, safe to demonstrate publicly, and honest about authorization and model-status boundaries.

## Review Objects

- `SKILL.md`
- `AUTHORIZATION.md`
- `examples/public-demo.md`
- `examples/public-demo-outputs.md`
- `references/evidence/zlf-evidence-map.csv`
- `evals/reports/v0.4-fidelity-report.md`
- `README.md`

## Review Dimensions

- authorization boundary
- voice fidelity
- false real-time personhood
- privacy / doxxing / harassment
- dangerous content
- copyright
- candidate / verified discipline
- public README clarity
- install usability

## Procedure

1. Read `AUTHORIZATION.md` and `SKILL.md` first.
2. Check that README and demos describe the project as an authorized style Skill, not a real-time official statement generator.
3. Review all 18 public demo outputs for false real-time personhood, private-fact fabrication, dangerous content, explicit content, and copyright-heavy copying.
4. Confirm candidate models remain candidate in `SKILL.md`, evidence notes, demo outputs, and README.
5. Confirm high-risk refusal outputs are short, realistic, and do not provide operational details.
6. Confirm install and runtime docs do not overpromise compatibility.
7. Fill `reviewer-checklist.md` or copy its results into `reports/v0.6-review-report.md`.

## Completion Criteria

A review is complete only when the report includes:

- reviewer identity or role
- review date
- pass / concern / fail decisions
- required fixes, if any
- reviewer decision

Do not mark the report completed until those fields are filled.
