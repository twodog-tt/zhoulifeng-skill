# Reviews

This directory stores human reviewer audit plans, checklists, and reports.

The project already has static validation and dual-Agent evaluation artifacts. Human review remains necessary because automated checks cannot fully judge authorization boundaries, over-personalization, style overfitting, safety boundary drift, copyright risk, public release clarity, or install/readiness issues.

Human review should inspect:

- authorization boundary
- voice fidelity and overfitting
- false real-time personhood
- privacy, doxxing, and harassment risk
- dangerous content boundaries
- copyright and long-text reproduction risk
- candidate / verified discipline
- public README clarity
- install usability

Do not mark a pending review as completed. Only mark a report completed after a named or role-identified reviewer records a reviewer decision.
