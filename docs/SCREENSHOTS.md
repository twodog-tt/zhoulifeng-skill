## Screenshot Safety Rules

Do not include screenshots that expose:

- private authorization materials
- private chats
- API keys, tokens, cookies, or credentials
- local private paths or personal directories
- non-public source files
- unpublished contracts, signatures, phone numbers, addresses, or private identities
- raw long subtitles, transcript dumps, or copyrighted source text
- private data about real interviewees

Before committing screenshots, crop or redact anything unrelated to public installation, validation, or demo usage.

Do not add fake screenshots. If a screenshot is not captured from a real public-safe flow, leave this document as the placeholder.
